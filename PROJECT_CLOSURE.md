Bus Reservation System – Project Closure Report
Project Overview
This project was developed as a final requirement for our coursework. Our team built a functional bus reservation system that allows users to view schedules and book tickets online.

 Repository Structure:
  Documentation:  This repo (Project overview and setup)
 	Front-End:  `BusTicketFrontEnd` (User Interface)
  Back-End:  `BusReservationBackend` (API and Database)

Key Achievements
Full Integration:  The React front-end successfully communicates with the Spring Boot/Node backend.
Database Management:  Created a working schema to store user data and ticket bookings.
Responsive UI:  The website works on both laptops and mobile browsers.
Project Goals:  All core requirements from the assignment rubric have been met.

 Lessons Learned
Version Control:  Using Git helped us manage code changes without breaking the project.
API Design:  We learned how important clear documentation is when connecting the front-end to the back-end.
Time Management: Breaking the project into "sprints" helped us finish before the deadline.
